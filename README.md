## About

Most of the things I build come from practical engineering problems.

`complexipy` started because cognitive complexity analysis was too slow and too disconnected from local development workflows. So I built a faster local-first alternative in Rust.

That same mindset usually pulls me into:

- static analysis
- developer infrastructure
- CI/CD tooling
- Python acceleration
- AI-assisted workflows
- OSS products
